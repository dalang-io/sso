The open-source Keycloak alternative.

A self-hosted OAuth 2.0 / OpenID Connect provider in Rust — your own “Sign in with…”, on your domain. Multi-tenant RBAC, post-quantum tokens, SDKs for six languages, and airgap-ready. One 7 MB binary, not a JVM.

Get started Download binary ★ Star on GitHub
🔑

OAuth 2.0 & OIDC

Authorization Code + PKCE, refresh rotation, client credentials, discovery, JWKS, UserInfo. Standards-compliant.

🏢

Multi-tenant RBAC

Isolated tenants with super / manager / developer roles. Managers own their clients; developers only rotate secrets.

🔐

Post-quantum ready

Optional ML-DSA (FIPS 204) token signing. Argon2 + SHA-256 hashing. Hybrid ML-KEM at the TLS edge.

📦

SDKs for 6 languages

JavaScript/TypeScript, Rust, Go, Python, Java, PHP — PKCE, token exchange, refresh, UserInfo, all consistent.

🗄️

Any database

SQLite by default (zero setup). PostgreSQL, MySQL & MariaDB by config. No recompile to switch.

✈️

Airgap & tiny

No CDN, no external fonts, one static binary with an embedded dashboard. Runs where the internet doesn’t reach.

Dalang SSO vs. Keycloak

Both are open-source identity providers. Dalang trades breadth for a small, fast, operable core.

Running in 60 seconds

# download the latest release binary (or `cargo build --release`) curl -L https://github.com/dalang-io/sso/releases/latest/download/sso-linux-x86_64 -o sso chmod +x sso && ./sso # open http://localhost:8080 → create your super admin → add an OAuth client

Then drop an SDK into your app and run the Authorization Code + PKCE flow against your instance. Full walkthrough in the docs.
